Mobile App Wireframes

End-to-end wireframe coverage for the TotalMedia Aware iOS/Android app. Screens are designed around two capability tiers: Basic (motion-only alerts) and Edge AI add-on (person, vehicle, intrusion, PPE detection). All screens follow the XProtect-inspired dark UI system with a four-tab bottom navigation.

Views — grouped by site/trailer
Each deployment site is collapsed into a card showing its trailer ID, online status, and a 2×2 camera grid. Designed for field supervisors managing multiple concurrent sites.
  • Sites grouped by trailer ID (e.g. #T-0042)

  • Edge AI detections show a red label overlay on the thumbnail (e.g. "Person detected")

  • Alarm bell icon in nav carries a red badge dot when unread alarms exist

Playback — timeline only
Historical footage playback with a scrubable timeline. Focused on the core review workflow for MVP — no live PiP overlay in this release.

Screen 07
Notification preferences — drill-down
Granular per-alert-type notification control. Edge AI alert types are labelled "requires add-on" to reflect the add-on-aware design principle from the MRD.
  • Per-type toggles: Person / Vehicle / Intrusion / PPE / Basic motion / Camera offline

  • Quiet hours: enable toggle, From/To time pickers, Critical override
  • "Override for Critical" ensures life-safety events still push during quiet hours

Screen 08
Change password — drill-down
In-app password change form for non-SSO accounts. The submit button remains disabled until all password rules pass, preventing invalid submissions.
  • Three fields: current password, new password (active), confirm new password
  • Blinking cursor on the active field indicates keyboard focus state
  • Real-time password rule checklist: green dot = passed, grey dot = not yet met
  • "Update password" button is greyed out until all four rules are satisfied
  • "Forgot current password?" link sends a reset email — no in-app reset flow
  • Entry point: Password row in the Security section of Profile
  • For SSO-managed accounts, the Password row should link to the operator portal instead
